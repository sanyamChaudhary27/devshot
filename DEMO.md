# SkillTrials Demo - Three Minutes, No Improvisation

## Objective

Show one clear transformation: trusted source material becomes a playable, testable decision model. The recording should show working product states, not a narrated architecture diagram or a live prompt experiment.

## Before recording

- Use the final deployed origin in a clean browser profile.
- Verify the public sample reaches its debrief and the author flow has a prepared, already-validated draft.
- Keep one short, authorized source ready for the generation segment. Do not rely on a model response arriving during the recording; capture a successful generation pass separately if necessary.
- Use a published public link for the signed-out segment.
- Turn off notifications, close unrelated tabs, and record at a readable zoom level.

## Storyboard

| Time | Screen and action | Narration |
| --- | --- | --- |
| 0:00-0:15 | Landing page, then enter the public sample. | "Policies tell people what to do. SkillTrials lets them prove they can apply it under pressure." |
| 0:15-0:45 | Show the situation, evidence, two metrics, and make one consequential choice. | "This is not a quiz. Each option changes a typed scenario state, so the learner sees the cost of a decision." |
| 0:45-1:05 | Reach the sample debrief and point to evidence labels. | "The player gets a deterministic score and a debrief grounded in the supplied material. The sample needs no model call." |
| 1:05-1:30 | Switch to author workspace. Show a prepared source and its source spans; start or show a completed generation. | "For authors, GPT-5.6 turns bounded source text and a brief into a structured scenario - not executable code." |
| 1:30-1:55 | Show validation/proof: citations, graph/outcome checks, and semantic review. | "A separate GPT-5.6 verifier checks semantic support. Deterministic code rejects invalid citations, weak choices, unreachable paths, and unsafe graphs; it permits one targeted repair." |
| 1:55-2:20 | Preview the generated draft and choose a different branch. | "Once validated, the fixed runtime plays every branch without another model call. This makes the result inspectable and repeatable." |
| 2:20-2:42 | Publish, then open the public link signed out. | "Private sources remain private. The share link exposes only the published trial, while verbatim private excerpts stay with the author." |
| 2:42-3:00 | Return to the proof/debrief or final public screen. | "SkillTrials compiles trusted material into a testable decision model: active practice, visible consequences, and evidence before publication." |

## Required evidence on screen

- One decision visibly changes state or outcome.
- One completed debrief with citations.
- One author proof/validation view.
- One public signed-out share page with private-source excerpts redacted.
- The distinction between GPT-5.6 generation/review and deterministic playback.

## Recovery plan

- If live generation fails or is slow, use the prepared validated draft and say it was generated from the displayed authorized source before recording.
- If authentication fails, continue with the public sample and public share; do not enter secrets or edit the database on camera.
- If the deployment fails, stop and fix the deployment before recording. A local-only recording must not be presented as a deployed workflow.

## Do not claim

- That trial scores predict job performance or learning outcomes.
- That citations prove a source is correct; they prove traceability to the supplied material.
- That the model runs the player or makes scoring decisions during play.
- Any user, accuracy, performance, or impact metric that has not been measured and documented.
